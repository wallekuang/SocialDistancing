Update History

V1.1.0 / 08-Aug-2020

Main Changes
  • ADV interval API called in case of "LOW_POWER_FEATURE" disabled
  • Scan interval , scan window and adv interval are kept same in high probability detection mode as per feedback
  • Optimized LED APIs. White led ON before sleep to avoid any interfernce of other led state
  • MAC generation from Unique serial number
  • Optimized adv and scan parameters to reduce the packet loss
  • Bug Fixed :
    • Before putting the device in sleep state due to accelerometer, device state has been modified from "NoMovementDetection" to "SleepMode" state, hence device goes in sleep and wakes up after 1 sec.
    • Device does not go in ideal state after waking up from sleep state
    • "DeviceState" was configured multiple times in NoMovementDetection state
    • Stop advertising and scanning before stopping virtual timer in NoMovementDetection state
    • Removed systick configuration : SysTick->CTRL = 0 and Clock_Init()

V1.0.0 / 01-June-2020

Main Changes
  • Low Power Mode : By default this feature is enabled. User can disable this by using "LOW_POWER_FEATURE" macro defined in "user_config.h" file.
  • Device continuously checks for Beacon. If there is no beacon for 2sec then device will switch in sleep mode for 1sec.
  • If no movement is detected from accelerometer for 60sec, it is assumed that user has removed the band and the device is put in the sleep mode until the device is tilted again.
  • Red Zone Detection: Alert condition is derived using Lookup Table. If any two or more devices comes in vicinity and fulfill the required condition an alert is generated in the form of Red Led . By default this feature is enabled. User can disable by using "RSSI_REDZONE" macro defined in "user_config.h" file.
  • Firmware supports two RSSI filtering techniques : a) Weighted mean filter b)Moving Average filter
  • Configured an RGB LED which will indicate whether the user is in safe zone or danger zone. If range is less than 2m, then Red LED will toggle and indicate that the user is in danger zone. If the range is more than 2m, then Green LED will toggle indicating safe zone.
  • Application debug messages will display on HyperTerminal if "DEBUG" macro is enabled .
